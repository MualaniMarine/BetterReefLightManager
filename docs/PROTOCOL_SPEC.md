# 协议说明

## 1. 当前范围

这份文档只描述当前已经在新项目里确认并使用的设备控制协议，不再混入过时推测。

协议实现主要在：

- [CommandUtil.kt](/D:/Dev/CodeX/NP/app/src/main/java/com/nemo/caideng/util/CommandUtil.kt)
- [ConnectThread.kt](/D:/Dev/CodeX/NP/app/src/main/java/com/nemo/caideng/ConnectThread.kt)

## 2. 基础连接

- 传输方式：TCP
- 目标端口：`8266`
- 连接后直接发送二进制报文
- 当前未发现额外握手包

## 3. 报文格式

### 发送包

通用格式：

`AAA5 + 功能码 + 数据区 + BB`

含义：

- `AAA5`：包头
- 功能码：2 字节
- 数据区：按功能不同变化
- `BB`：包尾

### 接收包

当前已确认两类：

- 短结果包
- 长配置读取包

用于识别读取响应的前缀：

- `ABAA`

## 4. 已确认功能码

| 功能 | 功能码 | 说明 |
|---|---|---|
| 同步时间 | `1003` | 下发当前时间 |
| 模式切换 | `1004` | 自动 / 手动切换 |
| 手动亮度 | `1005` | 下发当前 6 路亮度 |
| 预览亮度 | `1006` | 旧代码存在，当前 UI 未主用 |
| 全量配置下发 | `1007` | 下发手动亮度 + 曲线 + 状态 |
| 全量配置读取 | `1008` | 读取设备配置 |
| 演示模式 | `100A` | 开 / 关演示模式 |

## 5. 已实现指令

### 5.1 读取全部配置 `1008`

固定报文：

`AAA51008BB`

这是当前最稳定的联调入口。

### 5.2 同步时间 `1003`

格式：

`AAA5 1003 HH MM SS BB`

示例：

`AAA510030E2332BB`

表示：

- 14 时
- 35 分
- 50 秒

### 5.3 切换自动模式 `1004`

- 自动模式：`AAA5100401BB`
- 手动模式：`AAA5100400BB`

这里发送的是“明确切到某个模式”，不是切换开关。

### 5.4 手动亮度 `1005`

格式：

`AAA5 1005 V1 V2 V3 V4 V5 V6 BB`

说明：

- 共 6 路亮度
- 当前 UI 输入层使用十进制 `0-100`
- 封包发送时转成单字节值

### 5.5 演示模式 `100A`

- 打开演示模式：`AAA5100A00BB`
- 关闭演示模式：`AAA5100A01BB`

### 5.6 全量配置下发 `1007`

结构：

`AAA5 + 1007 + 手动亮度6字节 + 时间点数量 + 24组时间曲线 + 模式标记 + 当前时间 + BB`

当前工程里这条由 `CommandUtil.allSet(...)` 生成。

## 6. 时间曲线结构

当前约定 24 个时间点，每个时间点 8 字节：

| 顺序 | 含义 |
|---|---|
| 1 | 小时 |
| 2 | 分钟 |
| 3 | 第 1 路亮度 |
| 4 | 第 2 路亮度 |
| 5 | 第 3 路亮度 |
| 6 | 第 4 路亮度 |
| 7 | 第 5 路亮度 |
| 8 | 第 6 路亮度 |

## 7. 当前业务约定

- `SPS / LPS / SL` 是三组预设时间-亮度曲线
- 它们不是设备运行模式
- 自动 / 手动模式通过 `1004` 控制
- 演示模式通过 `100A` 控制

## 8. 当前已知但未完全定稿的部分

- `1006` 的业务语义仍未完全回接
- 读取响应包里各字段偏移仍有继续校准空间
- 配网协议不在本文档范围内
  - `Airkiss`
  - `Esptouch`

## 9. 联调建议

现场调试时优先按这个顺序验证：

1. `1008` 读取是否通
2. `1003` 时间同步是否通
3. `1005` 手动亮度是否生效
4. `1004 / 100A` 模式类指令是否生效
5. 最后再联调整包 `1007`
